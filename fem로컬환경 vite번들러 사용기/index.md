# FEM 로컬환경 Vite 번들러 사용기

- **브랜치**: `feature/PUB-8172-vite_fem`
- **PR**: [frontencar-mobile #4823](http://git.encar.io/projects/FR/repos/frontencar-mobile/pull-requests/4823/overview)

---

## 1. 작업 이유

| # | 문제 |
|---|------|
| 1 | FEM 로컬 서버 **가동 시간이 길다** → 서버를 다시 시작하기가 부담됨 |
| 2 | JS/SCSS 변경 시 **반영까지 오래 걸림** → 저장 후 새로고침이 습관화됨 |

---

## 2. 왜 Vite인가

| # | 이유 |
|---|------|
| 1 | 현재 **모노레포 패키지**에서 이미 Vite 사용 중 |
| 2 | **로컬 서버** 정도는 Vite로 충분히 구동 가능할 것으로 판단 |

---

## 3. Vite의 장점 (로컬 개발 관점)

| # | 내용 |
|---|------|
| 1 | **필요한 모듈만** 번들링 → **초기 구동 시간 단축** (전체 번들링 X) |
| 2 | **변경분만 반영** → Webpack은 변경 시 전체 재번들, Vite는 변경된 파일만 처리 (**HMR**) |
| 3 | **캐싱**이 잘 되어 있어 반복 작업이 빠름 |

---

## 4. 작업 이슈

| # | 이슈 | 비고 |
|---|------|------|
| 1 | **JSX 파싱 syntax 에러** | — |
| 2 | **순수 esbuild만으로는 실행 불가** | Babel 추가로 해결 |
| 3 | **JSX가 아닌 파일** 변경 시 HMR 미동작 → **전체 새로고침** 발생 | — |
| 4 | **SCSS 번들링 오류** | `nth($sprite, 9)` 등 문법 이슈 |

**SCSS 예시 (오류 발생 구간)**

```scss
background: url('../../../assets/scss/spr/nth($sprite, 9)') $sprite-offset-x $sprite-offset-y no-repeat;
```

## 5. 논의 내용 (적용 시 우려사항)

| 우려 | 설명 |
|------|------|
| **util.scss 수정** | 문법/경로 등 수정 사항 존재 |
| **index.html 위치** | Vite는 [프로젝트 루트 기준 index.html](https://ko.vitejs.dev/guide/#index-html-and-project-root)을 사용 → 위치가 기존(CRA)과 다를 수 있음 |
| **환경 차이** | 로컬은 Vite, **프로덕션·테스트는 Craco** 빌드 → 두 환경 차이로 인한 문제 가능성 |

---

## 6. 저의 의견

| 항목 | 의견 |
|------|------|
| util.scss | 간단한 문법 변경 수준이라 **수용 가능** |
| index.html 위치 | 초반엔 혼란 있을 수 있으나 **Craco 빌드에는 영향 없을 것** |
| 두 환경 차이 | 같은 소스(JS 등)를 기준으로 빌드하므로 **큰 문제 없을 것** |
| 주의 | **환경에 따라 동작이 달라지는 라이브러리**를 쓸 경우 **크로스 체크 필요** |
| 전략 | **로컬 전용**이므로 문제가 크면 **폐기**해도 무방 |
